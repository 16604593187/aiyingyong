项目主题：个人知识库助手 "Second Brain"

选这个主题的原因是它天然契合每一层的技术演进——从最简单的对话开始，逐步加入知识检索、记忆、多步推理，最终变成一个真正有价值的个人 AI 助手。而且这个项目对你自己就有实际用处，学习动力会更强。

第一层产出：最简对话终端

这一层的项目形态是一个命令行对话程序。功能很简单：启动后可以和模型持续对话，支持流式输出，对话历史在本次运行中保持，退出后清空。

核心代码结构只有一个文件 chat.py，维护一个 messages 列表，每次用户输入后追加到列表，调用 API，把返回追加进去，循环往复。

这一层结束时你有了项目的骨架，一个能跑的对话程序。

第二层产出：语义相似度工具模块

在项目里新增一个 embeddings.py 模块。功能是：给定一段文本，返回它的向量；给定两段文本，返回它们的相似度分数。

这一层还不接入主对话流程，但你会写一个独立的测试脚本，把你自己的一些笔记或文章片段转成向量，手动计算相似度，验证"语义相近的内容确实距离更近"。这个模块后面会成为整个知识库的核心基础设施。

第三层产出：本地知识库存储层

新增 knowledge_base.py 模块，封装向量数据库的操作：添加文档、检索相关文档、删除文档。底层用 Chroma，数据持久化到本地磁盘。

这一层你会写一个 ingest.py 脚本，能把一个文件夹里的 txt/md 文件全部读取、切片、Embedding 后存入知识库。运行完之后你的笔记就都"活"在向量库里了。

第四层产出：知识库问答（核心 RAG 升级）

这是项目最重要的一次升级。修改 chat.py，让每次用户提问时先去知识库检索相关片段，把检索结果拼入 Prompt，再调模型生成答案。同时在回答末尾附上"参考来源"，告诉用户答案来自哪个文件的哪个片段。

这一层结束后，你的助手第一次真正"认识"你的笔记了。你可以问它"我之前记录的关于 RAG 的笔记里提到了什么"，它能给出有依据的回答。

新增 rag.py 模块负责检索+生成的完整流程，chat.py 调用它。

第五层产出：有记忆的持久化助手

这一层做两件事。

第一件是加入对话记忆：让助手能理解多轮追问中的指代关系，对话历史超过一定长度时自动摘要压缩，而不是截断。

第二件是加入长期记忆：对话过程中如果用户提到了关于自己的重要信息（职业、偏好、正在做的项目等），自动提取并存入一个 user_profile.json。每次启动时加载这个文件，注入 system prompt，让助手"记得"你是谁。

这一层结束后，助手不再是每次启动都失忆的状态，它开始有了"人格"和对你的了解。

新增 memory.py 模块，项目目录新增 user_profile.json。

第六层产出：具备工具调用能力的 Agent

这是项目最大的一次架构升级。把 chat.py 的核心逻辑改造成一个 ReAct Agent 循环，给它配备一组工具：

search_knowledge_base：搜索你的本地知识库（第四层已有）
add_to_knowledge_base：把新内容存入知识库（让助手能自我学习）
web_search：搜索互联网获取实时信息
create_note：在 data/ 目录下创建新笔记文件
calculate：执行数学计算
这样助手就能处理复杂任务了，比如"帮我搜索一下 LlamaIndex 最新的 release notes，总结关键变化，然后存入我的知识库"——它会自动拆解成：搜索网页 → 总结内容 → 调用 add_to_knowledge_base → 完成。

新增 agent.py 模块，tools.py 定义所有工具。

第七层产出：生产级优化与评估体系

这一层不新增功能，而是让整个系统变得更好、更可靠。

在检索侧引入混合检索（向量检索 + BM25 关键词检索结果融合）和 Reranking（用交叉编码器对召回结果重新排序），解决专有名词检索不准的问题。加入查询改写，在检索前先让模型把用户的模糊问题改写成更适合检索的形式。

在评估侧用 RAGAS 框架建立一个测试集，量化评估系统的检索准确率和答案质量，每次优化后跑一遍评估，用数据说话。

最终形态的能力

走完七层之后，这个助手能做到：记得你是谁、记得你们聊过什么、能检索你所有的笔记和文档、能上网获取实时信息、能自主完成多步任务、能把新学到的内容自动存入知识库。它是一个真正意义上会随着你的使用而成长的个人 AI 助手。
