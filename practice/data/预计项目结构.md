第一层：最简对话终端


plaintext

second-brain/
├── chat.py              ← 主入口，对话循环 + 历史维护
├── config.py            ← API Key、模型名称等配置
└── requirements.txt     ← 依赖：openai 或 httpx
这一层只有三个文件。config.py 单独抽出来是好习惯，避免 API Key 硬编码在业务逻辑里。

第二层：新增 Embedding 工具模块


plaintext

second-brain/
├── chat.py
├── config.py
├── embeddings.py        ← 新增：文本转向量、计算余弦相似度
├── scripts/
│   └── test_embeddings.py   ← 新增：验证语义相似度的实验脚本
└── requirements.txt     ← 新增依赖：numpy
scripts/ 目录专门放实验和工具脚本，不污染主项目逻辑。test_embeddings.py 是这一层的"毕业验证"脚本，跑完它你就能直观看到语义空间的效果。

第三层：新增知识库存储层


plaintext

second-brain/
├── chat.py
├── config.py
├── embeddings.py
├── knowledge_base.py    ← 新增：封装 Chroma 的增删查操作
├── ingest.py            ← 新增：批量导入文档到知识库
├── data/                ← 新增：存放你的笔记/文档原文
│   └── (your .md / .txt files)
├── db/                  ← 新增：Chroma 持久化数据目录（自动生成）
├── scripts/
│   └── test_embeddings.py
└── requirements.txt     ← 新增依赖：chromadb
data/ 和 db/ 分开是因为前者是你的原始内容，后者是向量化后的索引，两者职责不同。db/ 应加入 .gitignore，不需要提交到版本控制。

第四层：核心 RAG 升级


plaintext

second-brain/
├── chat.py              ← 修改：调用 rag.py 而不是直接调模型
├── config.py
├── embeddings.py
├── knowledge_base.py
├── rag.py               ← 新增：检索 + Prompt 拼装 + 生成的完整流程
├── ingest.py
├── data/
├── db/
├── scripts/
│   ├── test_embeddings.py
│   └── test_rag.py      ← 新增：测试 RAG 问答效果的脚本
└── requirements.txt
rag.py 是这一层的核心，它把 knowledge_base.py（检索）和模型调用（生成）串联起来，chat.py 只负责 IO 交互，不再关心内部逻辑。

├── ingest.py
├── data/

plaintext

second-brain/
├── chat.py
├── config.py
├── embeddings.py
├── knowledge_base.py
├── rag.py
├── retrieval.py         ← 新增：混合检索（向量 + BM25）+ Reranking
├── agent.py
├── tools.py
├── memory.py
├── ingest.py
├── evaluate.py          ← 新增：RAGAS 评估流程
├── data/
├── db/
├── memory/
│   ├── user_profile.json
│   └── sessions/
├── logs/
│   └── agent_runs/
├── evals/               ← 新增：评估相关目录
│   ├── testset.json         ← 手工标注的测试问答对
│   └── results/             ← 每次评估的结果报告
│       └── 2024-01-15.json
├── scripts/
│   ├── test_embeddings.py
│   ├── test_rag.py
│   └── test_agent.py
└── requirements.txt     ← 新增依赖：rank-bm25、ragas、sentence-transformers
retrieval.py 把原来 rag.py 里简单的向量检索升级为混合检索，rag.py 调用它而不是直接调 knowledge_base.py。evals/ 目录是这一层的核心产出，testset.json 是你手工准备的标准问答对，results/ 记录每次优化前后的量化对比。
